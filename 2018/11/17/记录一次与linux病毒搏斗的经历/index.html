solr

  1. 安装tomcat
    tar -zxf apache-tomcat-7.0.47.tar.gz
    mkdir /usr/local/solr
    cp apache-tomcat-7.0.47 /usr/local/solr/tomcat -r
  2. 部署solr
    cp /root/solr-4.10.3/dist/solr-4.10.3.war /usr/local/solr/tomcat/webapps/solr.war
    把/root/solr-4.10.3/example/lib/ext 目录下所有的jar包复制到solr工程中
    cp * /usr/local/solr/tomcat/webapps/solr/WEB-INF/lib/
  3. 创建solrhome
    Solrhome是存放solr服务器所有配置文件的目录
    cp -r /root/solr-4.10.3/example solr/usr/local/solr/solrhome
  4. 修改web.xml
  5. 配置中文分析器
    cp IKAnalyzer2012FF_u1.jar /usr/local/solr/tomcat/webapps/solr/WEB-INF/lib/
    cp IKAnalyzer.cfg.xml ext_stopword.dic mydict.dic /usr/local/solr/tomcat/webapps/solr/WEB-INF/classes
  6. 配置fieldType
    vim solrhome/collection1/conf/schema.xml

    以上就是搭建全过程
    从一开始笨拙的两天三天 到现在半天 熟练的让人心疼

当然天将降大任于斯人也，必然不会这么简单
搭建好两天服务器就登录不上去了，一开始以为自己的代码有问题，，
后来找了腾讯客服原来是有异常进程占满cpu导致无法登录
top kill掉以后天真的以为问题消失
结果这个叫zigw的东西再次启动，cpu再次崩溃
我也崩溃了
whereis zigw 删掉以后又会出现

继续找客服
客服的回答很随意
重装系统

和病毒僵持了一个礼拜无果，只好妥协

重装吧 大不了再配两天环境

配好之后第二天继续懵逼
zigw有死而复活了

浪费了那么多时间，因为这个节外生枝的问题

迷茫了一段时间，，我知道这个项目结果不重要，，辉煌的是其中积累的经验和知识

后来不甘心，重装了系统 ，改了linux密码，数据库密码，tomcat密码
结果配置到一半的时候，，熟悉病毒再次出现了

心疼一波自己，不知道当时是怎么崩溃的坚持下来的

找了很多原因，排除所有的可能 ，问题出在了redis上

查看了redis库

发现了一个叫weppon的key

武器

value是一段脚本，其中有段url会执行下载一个脚本

别问我怎么知道，头很铁

脚本中有个函数名
killnginnl
杀死黑鬼

看来是个种族歧视老外干的

redis缓存数据库默认没有密码

攻击的原理是从入侵redis一步一步破解到linux

删掉这个key之后
以为问题消失了

然而还是太天真了

它攻击一次怎么可能没有第二次

这个叫weppon的key又出现了 带着它脚本

庆幸 ，找到了问题的源头

该设密码了

改redis.conf 开启
requirepass 密码

jedis.xml用jedispool的构造函数

重新发布，，问题解决

linux安稳运行
